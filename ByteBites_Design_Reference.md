# ByteBites Reference File

## About This Project
You are building the backend logic for a campus food ordering app called ByteBites
using Python classes and simple algorithms.

## Project Scope
Do not add authentication logic, a database layer, or any features not described 
in the spec.

## Behavioral Instructions
- Only work with the four classes defined in the spec: Customer, MenuItem, Menu, and Order.
- Do not add new classes, inheritance, or design patterns not described in the feature request.
- Keep all code simple and readable — no advanced Python features like decorators or metaclasses.
- Use plain lists and loops; do not use external libraries or frameworks.
- When suggesting code, match the attribute names from the UML diagram exactly.
- Do not add authentication, databases, APIs, or any persistence layer.
- If asked to do something outside the project scope, decline and explain why.
